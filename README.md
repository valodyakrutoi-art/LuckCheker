# Promo Checker

Веб-панель на Node.js: логинишься в Telegram-аккаунт (по номеру или по строке
сессии), вписываешь список промокодов (или грузишь `.txt`), бот сам шлёт их
`@LuckChecker_robot` с задержкой 3 сек, и если в ответе есть "Найдено
совпадение" — пересылает это сообщение в Избранное.

## Стек

- Express + Socket.io (веб-сервер и live-лог в браузере)
- [GramJS](https://gram.js.org/) (`telegram` npm-пакет) — MTProto-клиент,
  логинится в тот же Telegram, что и официальное приложение
- lowdb (JSON-файл) — хранит сохранённые сессии/номера. Это не полноценная
  SQL-база, а простой файл на диске, но так надёжнее с точки зрения
  деплоя (нет нативных зависимостей, которые могут не собраться на Railway).
  Чтобы данные не терялись при передеплое — храни файл на **Railway Volume**
  (см. ниже).

## 1. Получи api_id и api_hash

Раз они у тебя уже есть — просто держи их под рукой, понадобятся в переменных
окружения.

## 2. Локальный запуск (проверить, что всё работает)

```bash
npm install
cp .env.example .env
# впиши в .env свои TELEGRAM_API_ID и TELEGRAM_API_HASH
npm start
```

Открой `http://localhost:3000`.

## 3. Деплой на Railway

1. Залей этот проект в свой GitHub-репозиторий (или используй Railway CLI /
   "Deploy from local directory").
2. В Railway: **New Project → Deploy from GitHub repo**, выбери репозиторий.
3. Во вкладке **Variables** добавь:
   - `TELEGRAM_API_ID`
   - `TELEGRAM_API_HASH`
   - `TARGET_BOT_USERNAME` = `LuckChecker_robot` (можно не задавать — это
     значение по умолчанию)
   - `DATA_DIR` = `/data`
4. Добавь **Volume**: вкладка проекта → **Volumes → New Volume**, mount path
   `/data`. Это гарантирует, что сохранённые сессии не пропадут при
   передеплое.
5. Railway сам определит `PORT` — ничего дополнительно указывать не надо.
6. Деплой запустится автоматически. Открой сгенерированный Railway-домен —
   увидишь экран входа.

## Как это работает по шагам

1. **Экран входа**: по центру — выбор способа входа (стринг-сессия или
   номер + код + при необходимости пароль двухфакторки), справа — список уже
   сохранённых аккаунтов, можно кликнуть и сразу перейти дальше.
2. **Экран проверки**: вписываешь коды построчно в текстовое поле или
   загружаешь `.txt` (каждая строка = один код).
3. Жмёшь **"Начать проверку"** — сервер по очереди отправляет каждый код в
   личку `@LuckChecker_robot`, ждёт ответ (до 15 сек), проверяет, есть ли в
   нём фраза "Найдено совпадение":
   - если есть — пересылает это сообщение в Избранное и помечает в логе;
   - если нет — просто идёт дальше.
   - между сообщениями случайная пауза 3–5 секунд (каждый раз новое случайное
     значение в этом диапазоне — это менее предсказуемо для антиспам-систем,
     чем ровно 3 сек).
4. Весь прогресс стримится в браузер в реальном времени через Socket.io.
   Кнопка **"Остановить"** прерывает проверку после текущего сообщения.

## Важные нюансы

- **Скорость и лимиты.** Telegram может временно ограничивать аккаунт при
  слишком частой отправке сообщений одному боту. Задержка 3–5 сек (случайная
  каждый раз) — разумный минимум, не стоит её уменьшать, чтобы не словить
  `FLOOD_WAIT` или временный бан на отправку сообщений.
- **Один воркер = один аккаунт одновременно.** Сервер не ограничивает явно
  количество параллельных проверок, но лучше не запускать несколько джоб на
  одном аккаунте одновременно — это тот же Telegram-клиент.
- **Хранение сессий.** Строка сессии в lowdb — это фактически ключ доступа к
  твоему аккаунту, эквивалент пароля. Файл `data/db.json` не должен попадать
  в публичный репозиторий (он в `.gitignore`) и на общий Volume, к которому
  есть доступ у посторонних.
- Доступ к сайту без пароля — как и просил, ограничений на вход нет, значит
  ссылку на Railway-домен не стоит никому показывать.
